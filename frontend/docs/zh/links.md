# 链接与图谱

cardnote 卡片之间的关联通过**字段**而不是文本中的 `@` 提及来表达——这让关系结构化、可查询、可被规则驱动。

## 三种字段化的关联

### 1. `cardLink` 一对一

读书笔记上加一个"作者"字段（`cardLink`，目标合集 = 主题/人物）。填入一张人物卡片后：

- 读书笔记右侧面板显示作者名 + 头像，点击跳到人物卡
- 人物卡的「反向链接」面板自动出现这条读书笔记
- 双向同步——你不需要在人物卡上反过来再标记一次

### 2. `cardLinks` 一对多

人物卡上的"代表作品"字段（`cardLinks`，目标 = 主题/书籍 + 主题/影视）可以同时挂多张卡片。

### 3. `collectionLink`：卡片 → 合集

少见但实用——表达"这张卡片和某个合集相关"，但不改变它的归属合集。

## 反向链接

每张卡片的详情面板底部有「反向链接」区域，列出所有**通过 cardLink/cardLinks 字段**指向这张卡片的其它卡片。点击直接跳转，无需手动维护双向关系。

## 图谱视图（连接图）

侧栏的「连接图」入口打开一个全画布的图谱视图。

支持：
- **拖拽节点**调整位置
- **滚轮 / 双指**缩放
- **空白处拖拽**平移画布
- **点节点**跳到对应卡片
- 按 `objectKind`（卡片类型）筛选显示哪些节点
- 显示卡片间所有的 cardLink 关系作为连线

布局算法是**正交路由**（带圆角直角的连线）而不是力学引擎——节点位置可以手动调整并保存。当卡片数量增多时图谱仍保持稳定布局。

## autoLinkRules：自动建链

部分预设合集（如视频笔记）的 schema 里内置了 `autoLinkRules`——保存卡片时，根据某个字段值自动在两张卡片之间生成 cardLink，无需手动操作。详见 [合集模板](/docs/templates#autolinkrules自动建链)。

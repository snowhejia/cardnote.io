# 导入与迁移

cardnote 当前支持从 4 个来源批量导入历史笔记。所有导入都在浏览器本地解析，导入完成后才会写入服务端。

## 支持的来源

| 来源 | 文件格式 | 注意事项 |
|---|---|---|
| **Apple Notes** | iCloud 网页版导出的 ZIP / TXT | 含图片需用 ZIP；Markdown 风格的格式会保留 |
| **Evernote** | `.enex` | 标签变成 `choice` 字段；附件作为 file 卡片导入 |
| **Flomo** | 官方导出的 ZIP | tag 转为 `choice` 字段 |
| **语雀（Yuque）** | 知识库导出的 ZIP | 文档树结构映射为合集嵌套；图片自动转链 |

## 怎么导入

侧栏底部「⋯」菜单 → 选择对应导入器 → 拖入文件 → 预览（可勾选忽略哪些）→ 确认。

导入会在你账号下新建一个**导入合集**，所有内容都进这里——不会污染现有合集，可以慢慢手动整理或者直接整体保留。

## 不支持的来源

目前**没有**官方导入器：

- Notion（导出的 ZIP 含 HTML / Markdown，结构复杂，未来会支持）
- Obsidian（Vault 文件夹，纯 Markdown，技术上简单但未做）
- Roam Research、Logseq

## 导出 / 备份

cardnote 暂未提供"一键导出"功能。当前的备份方案：

1. 数据存在远程账号（Postgres）下，定期由后端备份
2. 单张卡片可以直接复制正文 / 下载附件
3. 自助批量导出 API 还在做中

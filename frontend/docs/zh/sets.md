# 合集

合集是卡片的容器。

## 多重归属

**一张卡片可以同时属于多个合集。** 数据模型上是 `card_placements` 多对多表——同一张卡片在每个合集里都有一条独立的"摆放记录"，包含 `pinned`（是否置顶）和 `sort_order`（排序）。

举个例子，一张关于 Linear 的笔记可以同时出现在：

- 「应用 / 工具」（按主题）
- 「2026 上半年」（按时间）
- 「Sprint 3」（按项目）

里。在每个合集里它的位置和置顶状态可以独立设置。

## 加 / 移操作

- **加入另一个合集** · 卡片详情页 → 「添加到合集」选目标合集
- **从某个合集移除** · 卡片详情页 → 在归属合集列表里点「⋯ → 从此合集移除」。卡片本身不会删除，只是脱离这个合集
- **拖拽** · 直接把卡片拖进侧栏的另一个合集

## 散卡

不属于任何合集的卡片会落到一个特殊位置 `LOOSE_NOTES_COLLECTION_ID`——你在「全部笔记」时间线里看得到。当你从最后一个合集里移除卡片时，它自动变成散卡而不是被删除。

## 嵌套

合集可以任意深度嵌套：

```
读书笔记
├── 文学
│   ├── 中国文学
│   └── 外国文学
└── 技术
    ├── 编程
    └── 设计
```

子合集的卡片**会**在父合集的视图里聚合显示（侧栏角标也是子树合计）。

## 基本操作

- **新建** · 侧栏底部「+ 新建合集」
- **重命名 / 改图标** · 侧栏右键「合集设置」
- **添加子合集** · 父合集右键「+ 子合集」
- **删除合集** · 删除时会询问内容如何处理（移到回收站 / 移到指定合集 / 变成散卡）

## 预设合集

注册时系统会创建几个预设合集，对应卡片类型的根：

- 笔记、文件、主题、剪藏、任务、项目、开支、账户

它们和普通合集一样**可以删除、可以重命名**。但删除后再新建同名合集不会自动恢复 schema——预设的字段定义在卡片子模板上而非合集本身，删了合集子模板还在。

## 已归档

侧栏的「已归档」入口是一个**特殊视图**而非合集——任何被打过 `archivedAt` 的卡片都会出现在那里。归档不影响卡片的合集归属，只是在常规列表里隐藏。

## 回收站

删除的卡片进回收站，30 天内可以还原；之后系统清理任务会硬删除。

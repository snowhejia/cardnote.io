# 简介

cardnote 是为「**快速记录、合集整理、深度链接**」三件事做的卡片笔记工具。

## 三件 cardnote 想做好的事

### 1. 快速记录

打开就写，不用先选模板、挑分类。**标题是可选的**——你可以直接写正文（第一行会自动当作标题展示），也可以用 Markdown `#` 加一个明确的标题；剪藏、文件这类卡片自带专门的「标题」字段。任何内容——一段话、一张图、一个网页、一个待办——都用同一种「卡片」装下，按下回车就保存了。

### 2. 合集整理

卡片**可以同时属于多个合集**，按主题归一份、按时间再归一份都行；不属于任何合集时它就是「散卡」，不会丢。合集本身可以任意嵌套，子合集的卡片在父合集视图里聚合显示。每个合集还可以挂一份 [Schema](/docs/templates)，定义这个合集里的卡片应有哪些字段。

### 3. 深度链接

卡片之间通过 `cardLink` / `cardLinks` 字段建立**结构化的双向链接**——不是文本里的 `@` 提及，而是字段级的引用。被引用的卡片自动出现「反向链接」，整张知识网络可以在「连接图」入口里直接看到、拖动、筛选。

## 三个基本概念

- **卡片（Card）**：所有信息的最小单位。每张卡片有正文、属性、关联，并归属在一个或多个合集下。
- **合集（Set）**：卡片的容器。合集可以任意嵌套；一张卡片可以同时属于多个合集，也可以暂时不属于任何合集（散卡）。
- **关联（Link）**：通过 `cardLink` / `cardLinks` 属性字段建立的卡片之间的引用，配套有反向链接。

## 八个内置卡片类型

| 类型 | 用途 | 子模板数 |
|---|---|---|
| 笔记 | 文字、想法、日记 | 6 |
| 文件 | 图片 / 视频 / 音频 / 文档 / 其他 | 5 |
| 主题 | 人物、组织、地点、书籍、影视…… | 12 |
| 剪藏 | 网页 / 邮件 / 各家社交平台 | 14 |
| 任务 | 待办 / 日程 / 习惯 | 3 |
| 项目 | 在做 / 已归档 | 2 |
| 开支 | 日常 / 订阅 / 报销 | 3 |
| 账户 | 登录 / 银行卡 / 证件 | 3 |

详见 [卡片类型](/docs/cards)。

## 适合谁

- 习惯**结构化笔记**、想给笔记加字段的人
- 想把读书笔记 / 网页剪藏 / 待办 / 项目都收在同一个工具里的人
- 不需要重度协作、想要更轻、更安静的 Notion / Obsidian 替代

## 当前阶段

cardnote 目前是 **beta 公开测试**，功能在快速迭代中：

- 单人使用，无协作
- 数据存在远程账号下，可在任何浏览器登录使用
- 不是端到端加密——账户卡的密码字段是明文存储在你账号下的，敏感凭证请自行评估再放进来
